# <a name="use-the-hospital-emergency-response-mobile-app"></a>病院の緊急時対応モバイル アプリを使用する

病院のスタッフは、緊急時にサプライ チェーンを管理しながら、患者数の増加に対応する必要があります。 現場の職員は、病院の緊急時対応モバイル アプリを使用することで、人工呼吸器、スタッフの配置、退院待ち、および COVID-19 関連の患者に関するデータの表示や追加を迅速に行うことができます。

## <a name="prerequisites"></a>前提条件

モバイル アプリの使用を開始するには、デバイスのアプリ ストアを使用して、Power Apps Mobile をデバイスにダウンロードする必要があります。

- [**Power Apps Mobile**](https://powerapps.microsoft.com/downloads) を**ダウンロードします**
    - iPhone や iPad などの iOS を搭載した **Apple** デバイスの場合は、[**App Store**](https://aka.ms/powerappsios) を使用します
    - **Android** デバイスの場合は、[**Google Play**](https://aka.ms/powerappsandroid) を使用します
- ご自身の組織によって、病院の緊急時対応アプリが、[アプリのデプロイと構成](deploy-configure.md)に関する記事で説明されているようにデプロイおよび構成されていることを確認します。

Power Apps Mobile をインストールしたら、デバイスからアプリを開き、ご自身の会社の Azure Active Directory アカウントを使用してサインインします。 サインインすると、組織が自分と共有しているすべてのアプリを表示できます。 詳細については、[Power Apps モバイル デバイスのサインイン](https://docs.microsoft.com/powerapps/user/run-app-client#open-power-apps-and-sign-in)に関する記事をご覧ください。

## <a name="hospital-emergency-response-mobile-app"></a>病院の緊急時対応モバイル アプリ

![病院の緊急時対応アプリ](media/use/app-launcher.png)

病院の緊急時対応モバイル アプリは、自分の役割に該当するさまざまなアプリを含んだモジュール構造になっています。 開始するには、Power Apps Mobile から病院の緊急時対応モバイル アプリを開き、ご自分の **[Hospital system]\(病院システム\)** 、 **[Region]\(地域\)、[Facility]\(施設\)** を選択してから **[Next]\(次へ\)** を選択します。

> [!NOTE]
> 病院の緊急時対応モバイル アプリまたはそのコンポーネントのいずれかを "*初めて*" 起動する場合は、アプリによるご自分の "*Office 365 ユーザー*" プロファイルと "*位置情報*" の読み取りを許可することに同意するように求められます。 選択したアプリの使用を開始する前に、 **[許可]** を選択する必要があります。 詳細については、「[同意](https://docs.microsoft.com/powerapps/user/run-app-client#give-consent)」をご覧ください。

## <a name="app-components"></a>アプリのコンポーネント

![病院の緊急時対応モバイル アプリのコンポーネント](media/use/app-components.png)

病院の緊急時対応サンプル ソリューション アプリは、ユーザー エクスペリエンスを向上させるために複数のアプリで構成されています。 ご自分の役割によっては、**病院の緊急時対応アプリ**に 1 つ以上のコンポーネントが表示されることがあります。

- **Bed capacity (ベッド容量)** 
    <br> 認可されたベッド、ICU ベッド、その他のベッド容量データなど、ベッドに関する情報を収集します。

- **COVID-19 stats (COVID-19 の統計情報)** 
    <br> COVID-19 の検査中である患者数と、検査が陽性だった患者数についての状態を収集します。

- **Equipment (機器)** 
    <br> 人工呼吸器、NIPPV、PAPR などの機器に関する情報を追跡します。

- **Staff (スタッフ)** 
    <br> 患者数と、パートナー、割り当て済み、要請済み、未割り当てなどの RN 状態に関する情報を収集します。

- **Supplies (消耗品)** 
    <br> 在庫の追跡、管理、および予測をより効果的に行うために、主要な消耗品を追跡します。 

- **Staffing needs (スタッフ配置のニーズ)** 
    <br> 部署、役割、および緊急度ごとに人材の要請を収集します。

- **Discharge planning (退院の計画)** 
    <br> 患者の退院に関する状態と予測を収集します。

> [!NOTE]
> 既定では、次のアプリの情報は "*場所*" レベルで追跡できます。**COVID-19 stats (COVID-19 の統計情報)** 、**Equipment (機器)** 、**Staff (スタッフ)** 。 残りのアプリでは、既定で "*施設*" レベルで情報を追跡できます。 管理者は、必要に応じて、既定の追跡レベルを変更することができます。 詳細情報: [モバイル アプリの追跡レベルを管理する](configure-data-reporting.md#manage-tracking-level-for-mobile-apps)

## <a name="staff"></a>[Staff]

![[Staff]](media/use/staff.png)

登録されている看護師、患者、および機器に関する、その場所に固有のインベントリを送信します。 画面上の各フィールドに値を入力する必要はありません。 ソリューション データベースに保存する必要があるフィールドに数値を入力します。

たとえば、要請済みの登録されている看護師の数に 3 を追加する必要がある場合は、 **[Registered nurses on duty - Requested]\(勤務中の登録されている看護師 - 要請済み\)** フィールドに「3」と入力し、 **[Submit]\(送信\)** を選択します。 また、使用中の **[Registered nurses on duty - Assigned]\(勤務中の登録されている看護師 - 割り当て済み\)** を 6 に更新する必要もある場合は、 **[Registered nurses on duty - Requested]\(勤務中の登録されている看護師 - 要請済み\)** フィールドに「3」と入力し、 **[Registered nurses on duty - Assigned]\(勤務中の登録されている看護師 - 割り当て済み\)** に「6」と入力してから、 **[Submit]\(送信\)** を選択します。

変更を送信せずに**病院の緊急時対応アプリ**に戻る場合は、左上から **[Back]\(戻る\)** を選択します。 **[Submit]\(送信\)** ボタンを使用すると、入力した値が送信されます。

データを送信した後、**Staff (スタッフ)** アプリに戻り、 **[Track another]\(他を追跡\)** ボタンを使用して別のレコードを作成することもできます。 **[Home]\(ホーム\)** を選択すると、**病院の緊急時対応アプリ**に戻ります。

### <a name="fields-and-description"></a>フィールドと説明

| **オプション名**               | **説明**                                                                                   |
|-------------------------------|---------------------------------------------------------------------------------------------------|
| 場所                      | 選択した施設内の部屋、病棟、またはその他の専門の場所の名前と種類。 |
| Number of patients (患者数)            | 選択した場所の現在の患者の合計数。                                        |
| **Registered nurses on duty (勤務中の登録されている看護師)** |                                                                                                   |
| *パートナー*                    | 選択した場所に現在配置されている登録された看護師のパートナーの数。                             |
| "*Requested (要請済み)* "                   | 選択した場所にいる要請済みの登録された看護師の数。                                  |
| "*Assigned (割り当て済み)* "                    | 選択した場所に割り当て済みの登録されている看護師の数。                                    |
| "*Unassigned (未割り当て)* "                  | 選択した場所にいるタスクを割り当てられていない登録されている看護師の数。                    |
| **施設レベルの追跡** |                                                                                                   |
| 現在この施設に不足している必須介護要員の割合 (%)                  | **施設全体**の合計に対して現在不足している必須介護要員の割合 (パーセンテージ形式)。                    |

## <a name="supplies"></a>Supplies (消耗品)

![Supplies (消耗品)](media/use/supplies.png)

**Supplies (消耗品)** アプリを使用して、消耗品のインベントリを表示します。 このアプリから、施設のインベントリ全体における消耗品コンポーネントの数と、1 日あたりのバーン レートを更新できます。

> [!NOTE]
> **[Submit]\(送信\)** を選択する前に、 **[In stock]\(在庫数\)** と **[Used past 24 hr]\(過去 24 時間に使用\)** の両方のフィールドに値を入力してください。

変更を送信せずに**病院の緊急時対応アプリ**に戻る場合は、左上にある **[Back]\(戻る\)** を選択します。 **[Submit]\(送信\)** ボタンを使用すると、入力した値が送信されます。 送信後は、 **[Home]\(ホーム\)** を選択すると、**病院の緊急時対応アプリ**に戻ります。

### <a name="fields-and-description"></a>フィールドと説明

消耗品アプリの項目の一覧は、組織の要件によって異なる場合があります。 消耗品の名前に関する説明については、ご自分の組織のリソースを参照してください。

IT 管理者は、Power Apps 用のモデル駆動型アプリを使用して、消耗品アプリの項目の一覧を追加または更新できます。 詳細については、[構成ガイド](deploy-configure.md)をご覧ください。

> [!NOTE]
> 消耗品在庫の項目の値は、数値形式である必要があります。

## <a name="staffing-needs"></a>Staffing needs (スタッフ配置のニーズ)

![Staffing needs (スタッフ配置のニーズ)](media/use/staffing-needs.png)

選択した施設に対する労働力の要請を収集します。 施設の労働力の要請を送信する前に、"*必須*" のマーク (*) が付いているフィールドが入力済みであることを確認してください。

変更を送信せずに**病院の緊急時対応アプリ**に戻る場合は、左上から **[Back]\(戻る\)** を選択します。 **[Submit]\(送信\)** ボタンを使用すると、入力した値が送信されます。 送信後は、 **[Home]\(ホーム\)** を選択すると、**病院の緊急時対応アプリ**に戻ります。

### <a name="fields-and-description"></a>フィールドと説明

| **フィールド名**           | **説明**                                                                            |
|--------------------------|--------------------------------------------------------------------------------------------|
| 部署               | 労働力を要請している部署の名前。 このフィールドは "*必須*" です。             |
| Department location (部署の場所)      | 部署の場所。                                                                |
| 要求の種類             | 要請の種類 (臨床、臨床以外など)。 このフィールドは "*必須*" です。 |
| Role needed (必要な役割)              | 要請された労働力の役割 (付き添い、登録された看護師など)。                          |
| Needed now or next shift (すぐに必要か次のシフトで必要か) | 要請された労働力に対するシフトを選択します (現在のシフト、または今後のシフト)。                |
| How many (必要数)                 | 必要なリソースの数 (数値形式)。                |
| 詳細                  | 労働力の要請に関するその他の詳細情報またはコメントを記述します。                        |

| Guardianship (保護)              | 保護中の患者の数。                             |

## <a name="other-options"></a>その他のオプション

このセクションでは、病院の緊急時対応モバイル アプリのコンポーネントを使って行えるその他のアクションについて説明します。

### <a name="end-shift---sign-out"></a>シフトの終了 - サインアウト

画面の左上にあるプロファイル アイコンを使用して、アプリからサインアウトできます。  

![サインアウト](media/use/sign-out.png)

**[End shift]\(シフトの終了\)** ボタンを選択すると、セッションを終了してサインアウトできます。

> [!NOTE]
> IT 管理者がデバイスの共有を無効にしている場合は、 *[End shift]\(シフトの終了\)* を使用できない可能性があります。

### <a name="app-feedback"></a>App feedback (アプリ フィードバック)

緊急時対応モバイル アプリの任意のコンポーネントから、**App feedback (アプリ フィードバック)** オプションを使用してフィードバックを共有できます。 フィードバックを共有するには、左上からご自分のプロファイルを選択して、 **[Submit feedback]\(フィードバックの送信\)** ボタンを選択します。

![フィードバックの提供](media/use/give-feedback.png)

**App feedback (アプリ フィードバック)** を選択すると、アプリについての良い点やアイデアを共有したり、問題を報告したりできます。

### <a name="switch-facility"></a>施設の切り替え

![施設の切り替え](media/use/switch-facility.png)

画面の右上にある施設名を選択して、いつでも施設を切り替えることができます。 場所の名前を選択すると、**病院の緊急時対応アプリ**の画面が表示され、そこで別の病院、地域、施設を選択できます。

## <a name="issues-and-feedback"></a>問題とフィードバック

- 病院の緊急時対応サンプル アプリに関する問題を報告するには、<https://aka.ms/emergency-response-issues> にアクセスしてください。

- 病院の緊急時対応サンプル アプリに関するフィードバックについては、<https://aka.ms/emergency-response-feedback> にアクセスしてください。
