# <a name="use-the-emergency-response-app"></a>緊急時対応アプリを使用する

病院のスタッフは、緊急時にサプライ チェーンを管理しながら、患者数の増加に対応する必要があります。 現場の職員は、緊急時対応モバイル アプリを使用することで、人工呼吸器、スタッフの配置、退院待ち、および COVID-19 関連の患者に関するデータの表示や追加を迅速に行うことができます。

## <a name="prerequisites"></a>前提条件

アプリの使用を開始するには、デバイスのアプリ ストアを使用して、Power Apps Mobile をデバイスにダウンロードする必要があります。

- [**Power Apps Mobile**](https://powerapps.microsoft.com/downloads) を**ダウンロードします**
- iPhone や iPad などの iOS を搭載した **Apple** デバイスの場合は、[**App Store**](https://aka.ms/powerappsios) を使用します
- **Android** デバイスの場合は、[**Google Play**](https://aka.ms/powerappsandroid) を使用します

Power Apps Mobile をインストールしたら、デバイスからアプリを開き、ご自身の会社の Azure Active Directory アカウントを使用してサインインします。 サインインすると、組織が自分と共有しているすべてのアプリを表示できます。 詳細については、[Power Apps モバイル デバイスのサインイン](https://docs.microsoft.com/powerapps/user/run-app-client#open-power-apps-and-sign-in)に関する記事をご覧ください。

## <a name="app-launcher"></a>アプリ起動ツール

![アプリ起動ツール](media/use/app-launcher.png)

緊急時対応モバイル アプリは、自分の役割に該当するさまざまなアプリを含んだモジュール構造になっています。 開始するには、Power Apps Mobile から緊急時対応モバイル アプリを開き、ご自分の **[Hospital system]\(病院システム\)** 、 **[Region]\(地域\)、[Facility]\(施設\)** を選択してから **[Next]\(次へ\)** を選択します。

> [!NOTE]
> 緊急時対応モバイル アプリ起動ツールまたはそのコンポーネントのいずれかを "*初めて*" 起動する場合は、アプリによるご自分の "*Office 365 ユーザー*" プロファイルと "*位置情報*" の読み取りを許可することに同意するように求められます。 選択したアプリの使用を開始する前に、 **[許可]** を選択する必要があります。 詳細については、「[同意](https://docs.microsoft.com/powerapps/user/run-app-client#give-consent)」をご覧ください。

## <a name="app-components"></a>アプリのコンポーネント

![緊急時対応モバイル アプリのコンポーネント](media/use/app-components.png)

緊急時対応サンプル ソリューション アプリは、ユーザー エクスペリエンスを向上させるために複数のアプリで構成されています。 ご自分の役割によっては、緊急時対応モバイル アプリの**アプリ起動ツール**に 1 つ以上のアプリが表示されることがあります。

- **Staff + equipment (スタッフと機器)** 
    <br> 施設内の場所別に RN と重要な機器の状態を収集します。

- **Supplies (消耗品)** 
    <br> 在庫の追跡、管理、および予測をより効果的に行うために、主要な消耗品を追跡します。 

- **Staffing needs (スタッフ配置のニーズ)** 
    <br> 部署、役割、および緊急度ごとに人材の要請を収集します。

- **COVID-19 stats (COVID-19 の統計情報)** 
    <br> COVID-19 の検査中である患者数と、検査が陽性だった患者数についての状態を収集します。

- **Discharge planning (退院の計画)** 
    <br> 患者の退院に関する状態と予測を収集します。

## <a name="staff--equipment"></a>Staff + equipment (スタッフと機器)

![Staff + equipment (スタッフと機器)](media/use/staff-equipment.png)

登録されている看護師、患者、および機器に関する、その場所に固有のインベントリを送信します。 領域の一覧は、**アプリ起動ツール**で選択された施設に固有のすべての場所で構成されます。 使用可能なオプションから場所を選択して、他のフィールドを更新します。

領域を選択したら、フィールドに必要な値を入力して、ソリューション データベースにレコードを保存します。 画面上の各フィールドに値を入力する必要はありません。 ソリューション データベースに保存する必要があるフィールドに数値を入力します。

たとえば、要請済みの登録されている看護師の数に 3 を追加する必要がある場合は、 **[Registered nurses on duty - Requested]\(勤務中の登録されている看護師 - 要請済み\)** フィールドに「3」と入力し、 **[Send]\(送信\)** を選択します。 使用中の人工呼吸器の数も 6 に更新する必要がある場合は、 **[Registered nurses on duty - Requested]\(勤務中の登録されている看護師 - 要請済み\)** フィールドに「3」と入力してから、 **[Equipment in use]\(使用中の機器\)** の下の **[Vents]\(人工呼吸器\)** に「6」と入力して、 **[Send]\(送信\)** を選択します。

変更を送信せずに**アプリ起動ツール**に戻る場合は、左上から **[Back]\(戻る\)** を選択します。 **[Send]\(送信\)** ボタンを使用すると、入力した値が送信されます。

### <a name="fields-and-description"></a>フィールドと説明

| **オプション名**               | **説明**                                                                                   |
|-------------------------------|---------------------------------------------------------------------------------------------------|
| 場所                      | 選択した施設内の部屋、病棟、またはその他の専門の場所の名前と種類。 |
| Number of patients (患者数)            | 選択した場所の現在の患者の合計数。                                        |
| **Registered nurses on duty (勤務中の登録されている看護師)** |                                                                                                   |
| *パートナー*                    | 選択した場所に現在配置されている登録された看護師のパートナーの数。                             |
| "*Requested (要請済み)* "                   | 選択した場所にいる要請済みの登録された看護師の数。                                  |
| "*Assigned (割り当て済み)* "                    | 選択した場所に割り当て済みの登録されている看護師の数。                                    |
| "*Unassigned (未割り当て)* "                  | 選択した場所にいるタスクを割り当てられていない登録されている看護師の数。                    |
| **Equipment in use (使用中の機器)**          |                                                                                                   |
| "*Vents (人工呼吸器)* "                       | 選択した場所で使用されている人工呼吸器の数。                                            |
| "*PAPR hoods (PAPR フード)* "                  | 選択した場所で使用されている電動ファン付き呼吸用保護具フードの数。                 |
| "*PAPR belts (PAPR ベルト)* "                  | 選択した場所で使用されている電動ファン付き呼吸用保護具ベルトの数。                 |
| "*PAPR chargers (PAPR 充電器)* "               | 選択した場所で使用されている電動ファン付き呼吸用保護具充電器の数。              |

## <a name="supplies"></a>Supplies (消耗品)

![Supplies (消耗品)](media/use/supplies.png)

**Supplies (消耗品)** アプリを使用して、消耗品のインベントリを表示します。 このアプリから、施設のインベントリ全体における消耗品コンポーネントの数と、1 日あたりのバーン レートを更新できます。

> [!NOTE]
> 送信を選択する前に、 **[In stock]\(在庫数\)** と **[Used past 24 hr]\(過去 24 時間に使用\)** の両方のフィールドに値を入力してください。

変更を送信せずに**アプリ起動ツール**に戻る場合は、左上にある **[Back]\(戻る\)** を選択します。 **[Send]\(送信\)** ボタンを使用すると、入力した値が送信されます。

### <a name="fields-and-description"></a>フィールドと説明

消耗品アプリの項目の一覧は、組織の要件によって異なる場合があります。 消耗品の名前に関する説明については、ご自分の組織のリソースを参照してください。

IT 管理者は、Power Apps 用のモデル駆動型アプリを使用して、消耗品アプリの項目の一覧を追加または更新できます。 詳細については、[構成ガイド](deploy-configure.md)をご覧ください。

**注:** 消耗品在庫の項目の値は、数値形式である必要があります。

## <a name="staffing-needs"></a>Staffing needs (スタッフ配置のニーズ)

![Staffing needs (スタッフ配置のニーズ)](media/use/staffing-needs.png)

選択した施設における労働力の要請を収集します。 施設の労働力の要請を送信する前に、"*必須*" のマークが付いているフィールドが入力済みであることを確認してください。

変更を送信せずに**アプリ起動ツール**に戻る場合は、左上から **[Back]\(戻る\)** を選択します。 **[Send]\(送信\)** ボタンを使用すると、入力した値が送信されます。

### <a name="fields-and-description"></a>フィールドと説明

| **フィールド名**           | **説明**                                                                            |
|--------------------------|--------------------------------------------------------------------------------------------|
| 部署               | 労働力を要請している部署の名前。 このフィールドは "*必須*" です。             |
| Department location (部署の場所)                 | 部署の場所。                                                                |
| 要求の種類             | 要請の種類 (臨床、臨床以外など)。 このフィールドは "*必須*" です。 |
| Role needed (必要な役割)              | 要請された労働力の役割 (付き添い、登録された看護師など)。                          |
| Needed now or next shift (すぐに必要か次のシフトで必要か) | 要請された労働力に対するシフトを選択します (現在のシフト、または今後のシフト)。                |
| Quantity (人数) | 必要な人数 (数値書式)。                |
| 詳細                  | 労働力の要請に関するその他の詳細情報またはコメントを記述します。                        |

## <a name="covid-19-stats"></a>COVID-19 stats (COVID-19 の統計情報)

![COVID-19 Stats (COVID-19 の統計情報)](media/use/covid19-stats.png)

 **COVID-19 stats (COVID-19 の統計情報)** アプリを使用して、COVID-19 固有の詳細情報を送信します。 その場所に固有の検査中の患者数と、陽性だった患者数を更新することができます。

また、 **[+ Add another location]\(+ 別の場所を追加\)** ボタンを使用して別の場所を追加し、複数の場所に関する統計情報を送信することもできます。

変更を送信せずに**アプリ起動ツール**に戻る場合は、左上から **[Back]\(戻る\)** を選択します。 **[Send]\(送信\)** ボタンを使用すると、入力した値が送信されます。

### <a name="fields-and-description"></a>フィールドと説明

| **フィールド名**  | **説明**                                                                                    |
|-----------------|----------------------------------------------------------------------------------------------------|
| 場所        | 選択した施設内の部屋、病棟、またはその他の専門の場所の名前と種類。  |
| PUIs (PUI)            | 検査中の患者の数。                                                            |
| Positive (陽性)        | COVID-19 陽性である患者の数。                                                         |

## <a name="discharge-planning"></a>Discharge planning (退院の計画)

![退院](media/use/discharge.png)

 **Discharge planning (退院の計画)** アプリを使用して、退院の情報と患者の状態を合計数で送信します。 過去 24 時間の退院の詳細情報、現在の退院の障壁、および障壁の分割を更新できます。

変更を送信せずに**アプリ起動ツール**に戻る場合は、左上から **[Back]\(戻る\)** を選択します。 **[Send]\(送信\)** ボタンを使用すると、入力した値が送信されます。

### <a name="fields-and-description"></a>フィールドと説明

| **フィールド名**            | **説明**                                                    |
|---------------------------|--------------------------------------------------------------------|
| 認証             | 承認プロセス中の患者の数。                   |
| Durable medical equipment (耐久医療機器) | 耐久医療機器を使用している患者の数。            |
| Guardianship (保護)              | 保護中の患者の数。                             |
| Home + Community Services (ホーム サービスとコミュニティ サービス) | ホーム サービスまたはコミュニティ サービスを使用している患者の数。               |
| 配置                 | 必要な配置の数。                                       |
| Skilled Nursing Facility (高度看護施設)  | 高度看護施設の数。                              |
| **Discharges (退院)**            |                                                                    |
| Past 24 h (過去 24 時間)                 | 過去 24 時間の退院が予想される患者の数。  |
| Likely next 24 h (おそらく今後 24 時間)          | 過去 24 時間に退院した患者の数。                    |
## <a name="other-options"></a>その他のオプション

このセクションでは、緊急時対応モバイル アプリのコンポーネントを使って行えるその他のアクションについて説明します。

### <a name="end-shift---sign-out"></a>シフトの終了 - サインアウト

画面の左上にあるプロファイル アイコンを使用して、アプリからサインアウトできます。  

![サインアウト](media/use/sign-out.png)

> [!NOTE]
> IT 管理者がデバイスの共有を無効にしている場合は、"*サインアウト*" を使用できない可能性があります。

### <a name="give-feedback"></a>フィードバックの送信

![フィードバックの送信](media/use/give-feedback.png)

緊急時対応モバイル アプリの任意のコンポーネントから、**App feedback (アプリ フィードバック)** オプションを使用してフィードバックを送信できます。 **App feedback (アプリ フィードバック)** を選択すると、アプリについての良い点やアイデアを共有したり、問題を報告したりできます。

### <a name="switch-facility"></a>施設の切り替え

![施設の切り替え](media/use/switch-facility.png)

画面の右上にある施設名を選択して、いつでも施設を切り替えることができます。 場所の名前を選択すると、**アプリ起動ツール**の画面が表示され、そこで別の病院、地域、施設を選択できます。
