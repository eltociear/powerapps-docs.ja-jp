# <a name="overview-of-the-canvas-app-for-northwind-traders"></a>Northwind Traders のキャンバス アプリの概要

[環境にインストール済](northwind-install.md) の Northwind Traders のデータベースでリレーショナル データを管理するためのキャンバス アプリについて説明します。 次に、続くトピックの詳細な手順に従ってこのアプリを最初から構築します。これにより、リレーショナル データを操作する実践的な経験を積むことができます。

このトピックでは、次のことを検出します:

- アプリ ユーザーがアプリでリレーショナル データを表示および管理する方法。
- アプリを起動するデータの種類。
- これらのデータの種類間での関連付けの作成方法。

単一画面で、アプリ ユーザーは受注を表示、更新、作成、削除できます。

> [!div class="mx-imgBorder"]
> ![キャンバス アプリの完了](media/northwind-orders-canvas-part1/orders-finished.png)

## <a name="explore-the-user-interface"></a>ユーザー インターフェイスの説明

### <a name="order-gallery"></a>受注ギャラリー

アプリの左端にあるギャラリーには、受注番号、状態、顧客の名前、受注合計金額を含む、受注の一覧が表示されます。 ユーザーは一覧をスクロールして受注を検索してから、受注の矢印を選択して、その受注に関する詳細情報を表示できます。 詳細: [受注ギャラリーの作成](northwind-orders-canvas-part1.md)。

### <a name="summary-form"></a>概要フォーム

右上隅にあるフォームは、ユーザーが受注ギャラリーで選択した受注についての概要です。 概要には、ギャラリーとほとんど同じ情報が含まれますが、受注の作成日および支払日、および受注を管理した従業員の名前と写真も表示されます。 ユーザーはタイトル バーの右端にあるアイコンを選択して、フォームのデータを変更したり、変更を保存したり、キャンセルしたり、受注を削除したりできます。 詳細: [概要フォームの作成](northwind-orders-canvas-part2.md)。

### <a name="detail-gallery"></a>詳細ギャラリー

右下隅のギャラリーには、選択した受注に含まれる製品とその数量に関する情報が表示されます。 このギャラリーの各項目は、受注明細と呼ばれます。 アプリ ユーザーはギャラリー内および下のコントロールを使用して、そのギャラリーにある任意の項目を追加および削除できます。 詳細: [詳細ギャラリーの作成](northwind-orders-canvas-part3.md)。

## <a name="explore-the-data-sources"></a>データ ソースの説明

このアプリを作成するには、5 つのエンティティとオプション セットからのデータを表示します。 実際にはこのアプリのほとんどの領域に、複数のエンティティからのデータが表示されます。 たとえば、受注ギャラリーには次の情報が含まれます:

- 受注番号は、**受注**エンティティのフィールドです。
- ステータスは、**注文**エンティティの別のフィールドで、**受注ステータス** オプション セットからのオプションです。
- 顧客名は、**顧客**エンティティのフィールドです。
- 合計コストは、**受注明細**エンティティのデータに基づいて計算されます。

概要には、受注の一覧と同じ情報が一部含まれますが、受注を管理した従業員の名前と写真も含まれています。 その情報は、**従業員**エンティティのフィールドから取得されます。 詳細ギャラリーには**受注明細**エンティティのレコードが表示され、それらの詳細の各製品は**受注製品**エンティティのレコードです。

## <a name="explore-the-relationships"></a>関連付けの説明

これらのエンティティにはデータベースで作成された関連付けがあるため、同じギャラリーまたはフォームの異なるソース (エンティティなど) からのデータを表示できます。

### <a name="many-to-one-relationships"></a>多対一関連付け

たとえば、各受注の顧客と従業員に関する情報は、**顧客**および**従業員**エンティティにあります。 したがって、**受注**エンティティには多数の受注があり、各受注は 1 人の顧客によってのみ発注され、1 人の従業員によってのみ管理されるため、これらのエンティティとの多対一の関連付けを持ちます。

各受注には、受注に含まれる製品とその数量を表す 1 つ以上の品目もあります。 各品目は**受注明細**エンティティのレコードで、**受注製品**エンティティから各製品に関する情報を取得します。 各明細では 1 つの製品のみを識別しますが、各製品は複数の明細に表示されます。 したがって、**受注明細**エンティティには**受注製品**エンティティと多対一の関連付けがあります。

### <a name="one-to-many-relationships"></a>一対多関連付け

各受注には複数の品目を含めることができますが、各品目は 1 つの受注のみに関連しています。 したがって、**受注**エンティティには**受注明細**エンティティと一対多の関連付けがあります。

### <a name="dot-notation-for-relationships"></a>関連付けのドット表記 

エンティティ間の関連付けに基づいてデータを表示するには、ドット プロパティ セレクターを使用して、あるエンティティから別のエンティティへの関連付けを移動できます。  たとえば、**受注**エンティティは、受注ギャラリーで顧客名を表示できるようにする**顧客**エンティティからの情報を取得します。 そのギャラリーでは、ラベルの **Text** プロパティを次の式に設定することにより、この動作を構成します:<br>`ThisItem.Customer.Company`

**ThisItem**では**受注**エンティティのレコードを指定し、注文した顧客に関する**顧客**エンティティから情報を取得します。 この場合、式では顧客の会社名が表示されることを指定します。 ただしその顧客のレコード全体が取得されるため、たとえば、その顧客の電子メール アドレスを代わりに簡単に表示できます。

あるエンティティから別のエンティティに移動する別の例として、ユーザーが別のギャラリーで選択したレコードと別のエンティティのレコードに基づいて、ギャラリーが 1 つのエンティティでレコードを表示するように指定できます。 受注明細を表示するには、詳細ギャラリーの **Items** プロパティを次の式に設定します:<br>`Gallery1.Selected.'Order Details'`

この場合、前の例でちょうど **ThisItem** が実行したように、**Gallery1.Selected** は**受注**エンティティのレコードを指定します。 ただし、この式は、前の式が実行したように 1 つのレコードのみを取得するわけではありません。 代わりに、レコードのテーブル全体を取得して、各製品の名前と単位あたりのコスト (**受注製品**エンティティに反映される) と数量 (**受注明細**エンティティに反映される) を表示します。
