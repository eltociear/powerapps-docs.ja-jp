# <a name="compose-a-page"></a>ページを作成する

必要な web ページを追加し、サイトマップで階層を管理した後、さまざまなコンポーネントを追加できます。 WYSIWYG エディターを使用すると、必要なコンポーネントをキャンバスに簡単に追加したり編集したりできます。 キャンバスには、次のコンポーネントを追加して編集できます。

- セクション
    - 1つの列セクション
    - 2つの列セクション
    - 3列セクション
- ポータルコンポーネント
    - テキスト
    - イメージ
    - IFrame
    - フォーム
    - 表
    - 階層リンク

> [!NOTE]
> PowerApps ポータル Studio を使用してポータルをカスタマイズした場合、web サイトのユーザーはパフォーマンスに影響します。 ライブポータルでは、ピーク時以外の時間帯に変更を行うことをお勧めします。 

## <a name="use-the-wysiwyg-editor"></a>WYSIWYG エディターを使用する

1.  [ポータルを編集](manage-existing-portals.md#edit)して PowerApps ポータル Studio で開きます。  

2.  コンポーネントを追加するページを選択します。

3.  キャンバスで編集可能な要素を選択します。

    > [!NOTE]
    > 編集可能な要素は、境界によって demarcated されます。

4.  画面左側の toolbelt から **コンポーネント**![コンポーネント アイコン](media/components-icon.png "コンポーネントアイコン")を選択します。  

5.  追加するコンポーネントを選択します。

    > [!div class=mx-imgBorder]
    > ![コンポーネントペイン](media/components-pane.png "コンポーネントペイン")  

    選択されたコンポーネントは、編集可能な要素内のキャンバスに追加されます。

6.  コンポーネントを削除するには、キャンバス上のコンポーネントを選択し、ページの上部にあるコマンドバーの **[削除]** を選択します。

    > [!div class=mx-imgBorder]
    > ![コンポーネントの削除](media/delete-component.png "コンポーネントの削除")  

## <a name="add-sections"></a>セクションの追加

セクションでは、ページの構造を定義し、それに応じてポータルコンポーネントを配置することができます。 ページにセクションを追加すると、要件に従ってセクション内にポータルコンポーネントを追加できます。

1.  [ポータルを編集](manage-existing-portals.md#edit)して PowerApps ポータル Studio で開きます。

2.  セクションを追加するページを選択します。

3.  キャンバスで編集可能な要素を選択します。

4.  画面左側の toolbelt から **コンポーネント**![コンポーネント アイコン](media/components-icon.png "コンポーネントアイコン")を選択します。

5.  **[セクションのレイアウト]** で、挿入するセクションの種類を選択します。

6.  画面の右側の [プロパティ] ウィンドウで、次の情報を入力または選択します。

    - **[高さの最小値]** : セクションの高さの最小値を入力します。 指定した高さよりも多くの領域を占有するコンポーネントを追加すると、コンポーネントに合わせてセクションが展開されます。 既定では、最小の高さは100px です。 高さをポイント (pt) およびパーセント (%) で入力することもできます。

        > [!div class=mx-imgBorder]
        > ![セクション内のアラインメント](media/section-props-height.png "セクション内のアラインメント")  

    - **Alignment**: セクション内のコンポーネントを左揃え、中央揃え、右揃えのいずれにする必要があるかを選択します。

        > [!div class=mx-imgBorder]
        > ![セクション内のアラインメント](media/section-props-align.png "セクション内のアラインメント")  

    - **Background**: セクションの背景として色または画像を含めるかどうかを選択します。

        - **塗りつぶし**: 背景色を選択します。

            > [!div class=mx-imgBorder]
            > ![セクションの塗りつぶしの色](media/section-props-fill.png "セクションの塗りつぶしの色")  

        - **[イメージ]** : 一覧からイメージを選択します。 新しいイメージをアップロードする場合は、[イメージの**アップロード**] を選択します。

            > [!div class=mx-imgBorder]
            > ![セクションに画像を追加します](media/section-props-image.png "セクションに画像を追加します")  

7.  セクションに必要なポータルコンポーネントを追加します。

### <a name="add-text-box"></a>テキスト ボックスの追加

1.  [ポータルを編集](manage-existing-portals.md#edit)して PowerApps ポータル Studio で開きます。  

2.  コンポーネントを追加するページを選択します。

3.  キャンバスで編集可能な要素を選択します。

4.  画面左側の toolbelt から **コンポーネント**![コンポーネント アイコン](media/components-icon.png "コンポーネントアイコン")を選択します。  

5.  **[ポータルコンポーネント]** で、 **[テキスト]** を選択します。

6.  テキストボックスに必要なテキストを入力します。

7.  テキストの書式を設定するには、テキストを選択して、書式オプションを表示します。 必要に応じて、フォントサイズとスタイルを変更します。

    > [!div class=mx-imgBorder]
    > ![テキストコンポーネント](media/text-component.png "テキストコンポーネント")  

8. 画面の右側の [プロパティ] ウィンドウで、次の情報を選択します。

    - **[配置]** : テキストを左揃え、中央揃え、右揃えのいずれにするかを選択します。

    - **[フォントの色]** : テキストの色を選択します。

        > [!div class=mx-imgBorder]
        > ![テキストの配置と色の選択](media/text-props.png "テキストの配置と色の選択")  
 

### <a name="add-image"></a>イメージの追加

1.  [ポータルを編集](manage-existing-portals.md#edit)して PowerApps ポータル Studio で開きます。  

2.  コンポーネントを追加するページを選択します。

3.  キャンバスで編集可能な要素を選択します。

4.  画面左側の toolbelt から **コンポーネント**![コンポーネント アイコン](media/components-icon.png "コンポーネントアイコン")を選択します。  

5.  **[ポータルコンポーネント]** で、 **[イメージ]** を選択します。 画像のプレースホルダーがキャンバスに追加されます。

6.  画面の右側の [プロパティ] ウィンドウで、次の情報を入力します。

    - **[イメージ]** : 既存のイメージを選択するか、新しいイメージをアップロードする場合は、このオプションを選択します。 アップロード済みのイメージを選択する場合は、 **[イメージの選択**] ボックスの一覧からイメージを選択します。 新しいイメージをアップロードするには、 **[イメージのアップロード]** を選択します。 アップロードされたすべてのイメージはイメージライブラリに含まれており、 **[イメージの選択**] ボックスの一覧から選択できます。

        > [!div class=mx-imgBorder]
        > ![画像のプロパティ](media/image-props.png "画像のプロパティ")  

        > [!NOTE]
        > - アップロードできるのは、png、svg、jpg、jpeg の種類のイメージのみで、最大サイズは 5 MB です。
        > - 同じ名前のイメージをアップロードすることはできません。 イメージの名前を変更して再度アップロードする必要があります。

    - **[外部 url]** : 外部 url からイメージをアップロードする場合は、このオプションを選択します。 **[外部 url]** フィールドに URL を入力します。 セキュリティで保護されたリンクのみが受け入れられます。つまり、 https://は必須です。 コンテンツ配信ネットワークに画像が格納されている場合は、このフィールドにリンクを指定できます。

        > [!div class=mx-imgBorder]
        > ![画像の外部 url](media/image-ext-url.png "画像の外部 URL")  

    -   **書式設定のオプション**

        - **Width**: イメージの幅を入力します。

        - **Height**: イメージの高さを入力します。

    > [!NOTE]
    > キャンバスで画像を選択し、ハンドルをドラッグしてサイズを変更することもできます。

### <a name="add-iframe"></a>IFrame の追加

1.  [ポータルを編集](manage-existing-portals.md#edit)して PowerApps ポータル Studio で開きます。  

2.  コンポーネントを追加するページを選択します。

3.  キャンバスで編集可能な要素を選択します。

4.  画面左側の toolbelt から **コンポーネント**![コンポーネント アイコン](media/components-icon.png "コンポーネントアイコン")を選択します。  

5.  **[ポータルコンポーネント]** で、 **[IFrame]** を選択します。 IFrame プレースホルダーがキャンバスに追加されます。

6.  画面の右側の [プロパティ] ウィンドウで、次の情報を入力します。

    - **Width**: IFrame の幅を入力します。

    - **高さ**: IFrame の高さを入力します。

    - **リンク**: IFrame に表示する web サイトの URL を入力します。 セキュリティで保護されたリンクのみが受け入れられます。つまり、 https://は必須です。 既定では、<https://www.bing.com> は値として使用できます。

        > [!div class=mx-imgBorder]
        > ![iframe のプロパティ](media/iframe-props.png "IFrame のプロパティ")  

    > [!NOTE]
    > また、キャンバスで IFrame を選択し、ハンドルをドラッグしてサイズを変更することもできます。

### <a name="add-form"></a>フォームの追加

フォームは、ポータルでデータを収集するためのフォームを追加するために使用するデータドリブン構成で、開発者がポータルでフォームを表示する必要がありません。 [フォームは Common Data Service で作成され](https://docs.microsoft.com/powerapps/maker/model-driven-apps/form-designer-overview)、ポータルの web ページ、またはリストと組み合わせて使用して、完全な web アプリケーションを作成できます。  

1.  [ポータルを編集](manage-existing-portals.md#edit)して PowerApps ポータル Studio で開きます。  

2.  コンポーネントを追加するページを選択します。

3.  キャンバスで編集可能な要素を選択します。

4.  画面左側の toolbelt から **コンポーネント**![コンポーネント アイコン](media/components-icon.png "コンポーネントアイコン")を選択します。  

5.  **[ポータルコンポーネント]** で **[フォーム]** を選択します。

6.  画面の右側の [プロパティ] ウィンドウで、次のいずれかのオプションを選択します。

    - **新規作成**: 新しいフォームを作成します。
    - **既存**のフォームを使用します。既存のフォームを使用します。

7. 次の情報を入力するか、選択します。

    - **名前**: フォームの名前。

    - **エンティティ**: フォームの読み込み元となるエンティティの名前。

    - **フォームレイアウト**: 表示される Common Data Service 内のターゲットエンティティ上のフォームの名前。

    - **モード**: 次のオプションのいずれかを選択します。

        - **Insert**: フォームが送信時に新しいレコードを挿入するかどうかを示します。

        - **Edit**: フォームが既存のレコードを編集する必要があることを示します。

        - **Read only**: フォームに、既存のレコードの編集不可フォームが表示されることを示します。

        > [!NOTE]
        > **編集**モードと**読み取り専用**モードの既定のオプションは、URL に ID として渡されるクエリ文字列パラメーター名として設定されます。 これらの値を変更するには、ポータル管理アプリを開き、フォームのプロパティを更新する必要があります。

    - **[成功時**]: 次のいずれかのオプションを選択します。

        - [成功した**メッセージの表示]** : フォームが正常に送信されたときに、ユーザーにメッセージを表示する必要があります。 また、[**成功時にフォームを非表示**にする] を選択すると、送信が成功したときにフォームが非表示になります。

        - **Web ページにリダイレクトする**: ポータルで選択した web ページにユーザーをリダイレクトします。 **[Web ページにリダイレクト]** の一覧から web ページを選択する必要があります。

        - **Url へのリダイレクト**: 指定した url にユーザーをリダイレクトします。 **[Url にリダイレクト]** フィールドに url を入力する必要があります。

    - **Show captcha for anonymous users**: 匿名ユーザーに captcha を表示します。

    - [ **Authenticated users の captcha を表示する]** : 認証されたユーザーに captcha を表示します。

    - **エンティティのアクセス許可を有効**にします。エンティティのアクセス許可をフォームに対して考慮する必要があります。 既定では選択されていません。 この設定を選択した場合、すべてのユーザーがフォームにアクセスするためには明示的な権限が必要になります。 詳細情報:[エンティティのアクセス許可](configure/assign-entity-permissions.md)

        > [!div class=mx-imgBorder]
        > ![フォームのプロパティ](media/form-props.png "フォームのプロパティ")

### <a name="add-list"></a>リストの追加

リストは、レコードの一覧を表示する web ページを追加するために使用するデータドリブン構成で、開発者がポータルにグリッドを表示する必要がありません。 リスト[Common Data Service ビュー](https://docs.microsoft.com/powerapps/maker/model-driven-apps/create-and-edit-views)を使用して、ポータルにレコードを表示します。  

1.  [ポータルを編集](manage-existing-portals.md#edit)して PowerApps ポータル Studio で開きます。  

2.  コンポーネントを追加するページを選択します。

3.  キャンバスで編集可能な要素を選択します。

4.  画面左側の toolbelt から **コンポーネント**![コンポーネント アイコン](media/components-icon.png "コンポーネントアイコン")を選択します。  

5.  **[ポータルコンポーネント]** で、 **[一覧]** を選択します。

6.  画面の右側の [プロパティ] ウィンドウで、次のいずれかのオプションを選択します。

    - **新規作成**: 新しいリストを作成します。
    - **既存のものを使用**: 既存のリストを使用します。

7.  次の情報を入力するか、選択します。

    - **[名前]** : リストの名前。

    - **エンティティ**: ビューの読み込み元のエンティティの名前。

    - **Views**: 表示するターゲットエンティティのビューの一覧。 複数のビューを選択して、一覧にレコードを表示することができます。 最初に選択したビューが既定のビューになります。

    - **新しいレコードの作成**: ユーザーがレコードを作成できるようにします。 新しいレコードを作成するフォームが含まれている web ページを選択します。

    - **詳細の表示**: ユーザーが詳細を表示できるようにします。 詳細を表示するフォームが含まれている web ページを選択します。

    - **レコードの編集**: ユーザーがレコードを編集できるようにします。 レコードを編集するフォームが含まれている web ページを選択します。

    - **レコードの削除**: ユーザーがレコードを削除できるようにします。

    - **空のリストメッセージ**: 表示するレコードがない場合に表示されるメッセージ。

    - [ **1 ページあたりのレコード数]** : ページに表示するレコードの数を指定する整数値です。

    - **[エンティティリストで検索を有効**にする]: ユーザーがリスト内のレコードを検索できるようにします。

    - **エンティティのアクセス許可を有効**にします。エンティティのアクセス許可を一覧に対して考慮する必要があります。 既定では選択されていません。 この設定を選択した場合、すべてのユーザーがフォームにアクセスするためには明示的な権限が必要になります。 詳細情報:[エンティティのアクセス許可](configure/assign-entity-permissions.md)  

    > [!div class=mx-imgBorder]
    > ![リストのプロパティ](media/list-props.png "リストのプロパティ")

### <a name="add-breadcrumb"></a>階層リンクの追加

1.  [ポータルを編集](manage-existing-portals.md#edit)して PowerApps ポータル Studio で開きます。  

2.  コンポーネントを追加するページを選択します。

3.  キャンバスで編集可能な要素を選択します。

4.  画面左側の toolbelt から **コンポーネント**![コンポーネント アイコン](media/components-icon.png "コンポーネントアイコン")を選択します。  

5.  **[ポータルコンポーネント]** で、 **[階層リンク]** を選択します。

## <a name="add-a-custom-menu"></a>カスタムメニューを追加する

既定では、web ページの階層に基づいて、web サイトのメニューが自動的に作成されます。 これは、**既定**のメニューと呼ばれます。 カスタムメニューを作成するには、ポータル管理アプリで web リンクセットを作成する必要があります。 詳細情報: [web リンクの管理](configure/manage-web-links.md)

Web リンクセットを作成したら、次のようにします。

1.  [ポータルを編集](manage-existing-portals.md#edit)して PowerApps ポータル Studio で開きます。

2.  ヘッダーコンポーネントを選択します。 

3.  画面の右側にある [プロパティ] で、**ナビゲーションメニュー**の一覧から web リンクセット名を選択します。

    > [!div class=mx-imgBorder]
    > ![ナビゲーションメニュー](media/navigation-menu.png "ナビゲーションメニュー")

## <a name="use-code-editor"></a>コードエディターを使用する

キャンバス上のコンポーネントのソースを表示するには、コンポーネントを選択し、[ソースコードエディター] アイコンを選択して、フッターの **&gt;/&lt;** します。

> [!div class=mx-imgBorder]
> ![コードエディターのアイコン](media/code-editor-icon.png "コードエディターのアイコン")  

ソースコードは、画面の下部にある**コードエディター**ペインに表示されます。 以前に行った変更は、ソースコードで更新されています。 変更するには、ソースコードを更新し、 **[保存]** を選択します。 変更はキャンバスに反映されます。

> [!div class=mx-imgBorder]
> ![コードエディター](media/code-editor.png "コードエディター") 

> [!NOTE]
> また、詳細な構成のために、ソースコードエディターで液体タグを追加することもできます。 詳細情報:[水冷テンプレートの使用](liquid/liquid-overview.md)
