#  <a name="basic-navigation-in-a-model-driven-app"></a>モデル駆動型アプリでのナビゲーションの基本 

この概要では、アプリを検索して開く方法と、リスト、フォーム、ビジネス プロセスなどの一般的なユーザー インターフェイス要素を操作する方法について説明します。

## <a name="navigating-among-apps-areas-and-entities"></a>アプリ、領域、およびエンティティ間の移動

モデル駆動型アプリは、アプリケーション (アプリ)、領域、およびエンティティから構築されています。

- "*アプリ*" では、アカウントや連絡先の管理など、特定のクラスのアクティビティを実現するための機能のコレクションが提供されます。 アプリセレクター メニューを使用して、組織で利用可能なアプリ間を移動します。

- "*作業領域*" はアプリの下位部分であり、特定の機能専用です。 各作業領域には、その領域で作業できるようにエンティティのターゲット コレクションが用意されています。 場合によっては、同じエンティティが複数の領域 (または複数のアプリ) に表示されることもあります。 たとえば、連絡先エンティティとアカウント エンティティは、さまざまなアプリと作業領域に表示されます。 作業領域メニューを使用して、現在のアプリの作業領域間を移動します。

- "*エンティティ*" は、連絡先やアカウントなど、特定の種類のデータを表します。 エンティティには構造化されたデータ形式が使用され、これにより、エンティティで使用できるフィールドのコレクションが定義されます。 各エンティティは、個々のレコードのコレクションで構成されます。 たとえば、連絡先エンティティの場合は、各レコードに 1 人の人物が記述され、各レコードには、名、姓、電子メール アドレスなどのフィールドのコレクションが含まれます。 通常、エンティティには 2 つのビューがあります。リスト ビューには通常、使用可能なレコードが一覧表示され、フォーム ビューには、1 つのレコードに対して使用可能なすべてのデータと設定が表示されます。 現在の作業領域内のエンティティ間を移動するには、サイド ナビゲーターを使用します。

### <a name="move-between-apps"></a>アプリ間を移動する

アプリを切り替えるには、アプリセレクター メニューを使用します。

![アプリセレクター メニュー](media/app-selector.png "アプリセレクター メニュー")

アプリセレクター メニューに表示されるアプリは、ユーザーがどのアプリにアクセスできるかによって異なります。 

### <a name="move-between-entities-records-and-work-areas"></a>エンティティ、レコード、および作業領域間の移動

お気に入りのレコードや使用頻度の高いレコードに簡単にアクセスして戻ることができます。 次の図は、プライマリ ナビゲーション要素を示しています。

![ナビゲーション コントロール、展開されたビュー](media/nav-expanded.png "ナビゲーション コントロール、展開ビュー")

凡例:

1. **アプリ セレクター**:このメニューを開いてアプリ間を移動します。
2. **折りたたみ/展開ボタン**:これを選択してナビゲーターを折りたたむと、ページの主要部分のスペースを広げることができます。 ナビゲーターが既に折りたたまれている場合は、このボタンを選択すると、再度展開します。
3. **最近使用したレコード**:このエントリを展開すると、最近使用したレコードの一覧が表示されます。 ここでレコードを選択して開きます。 ここに表示されているレコードの横にあるプッシュピン アイコンを選択して、お気に入り (ピン留めされたレコード) に追加します。
4. **お気に入りのレコード**:このエントリを展開してお気に入りの (ピン留めするされた) レコードを表示し、開きます。 **[最近使用したレコード]** 一覧を使用し、ここにレコードを追加します。 この一覧から削除するには、ここに表示されているレコードの横にあるピンの削除アイコンを選択します。
5. **エンティティ ナビゲーター**:この領域には、現在の作業領域で使用できる各エンティティとダッシュボードが一覧表示されます。 ここで任意のエントリを選択して、そのエンティティの名前付きダッシュボードまたはリスト ビューを開きます。
6. **作業領域セレクター**:このメニューを開いて、別の作業領域に移動します。 ここには、現在の作業領域の名前が表示されています。

## <a name="working-with-list-views"></a>リストビューの操作

通常、エンティティを初めて開くと、リスト ビューが表示され、そのエンティティに属するレコードの一覧が表形式で示されます。 たとえば、**アカウント** エンティティを開くと、アカウントの一覧が表示されます。

![一般的なリスト ビュー](media/list-view.png "一般的なリスト ビュー")

凡例:

1. **レコードの選択**:この列にチェック マークを付けて、1 つまたは複数のレコードを選択します。 作業中の場所によっては、コマンド バーのボタンを使用して、選択したすべてのレコードに一度に 1 つの操作を適用できます。
2. **レコードを開く**:リスト内の任意のレコードを選択すると、レコード ビューが開き、レコードの詳細がすべて表示されます。 通常、現在のエンティティからレコードを開くには、 **[名前]** 列から選択します。 エンティティによっては、他の列 (関連連絡先など) に関連エンティティのレコードへのリンクが提供されます。
3. **リストの並べ替えまたはフィルター処理**:選択すると、その列の値によって一覧が並べ替えられます。または、その列の値で一覧をフィルター処理できます。 列見出しの矢印は、どの列がどの方向に並べ替えられているかを示します。 
4. **コマンド バー**:コマンド バーのコマンドを使用して、一覧のレコードを操作し、関連するアクションを実行します。 一部のコマンド (**削除**など) では、最初に左端の列にチェック マークを付けて 1 つ以上のターゲット レコードを選択する必要があり、他のコマンドでは一覧全体を操作します。 作業しているレコードの種類に応じて、一覧を Excel ブックにエクスポートしたり (場合によってはテンプレートに基づいて)、グラフやダッシュボードを開いたりすることができます。
5. **ビューの検索**:一覧の上にある検索フィールドにテキストを入力すると、現在のビュー内でそのテキストを含むレコードのみが表示されます。
6. **フィルター処理とページング**:文字を選択すると、その文字で名前が始まるレコードのみが表示されます。 一覧に、1 ページに表示できるよりも多くのレコードが含まれる場合は、一覧の下部にあるページング矢印を使用して、ページを前後に移動します。

## <a name="working-with-record-views"></a>レコード ビューの操作

レコード ビューには、1 つのレコードに関するすべての詳細情報が表示され、そのレコードを操作するための特別な機能も提供されます。 通常は、リスト ビューに表示されるレコードを選択してレコード ビューを開きますが、関連レコードからリンクを使ってレコード ビューを開くこともできます。

![一般的なレコード ビュー](media/form-view.png "一般的なレコード ビュー")

凡例:


1. **タブ**:ほとんどのレコード ビューは、タブに分割されています。 各タブで、レコードの関連フィールドのコレクションが提供されます。 使用可能なタブが、レコード名の下に表示されます。 タブ名を選択して、そのタブにアクセスします。現在のタブは下線付きで表示されています。
2. **関連**:ほとんどすべての種類のレコードは、少なくとも 1 回保存すると **[関連]** タブに表示されます。 このタブは実際にはドロップダウン リストで、これを使用して、表示されているレコードを使用または参照する他の種類のレコードを検索できます。 **[関連]** ドロップダウン リストからエンティティ名を選択すると、そのエンティティの名前が付けられた新しいタブが開き、その種類のすべての関連レコードの一覧が表示されます。 **[関連]** タブは引き続き使用できるので、これを使用して、現在のレコードを参照している他の種類のレコードを見つけることができます。
3. **コマンド バー**:コマンド バーのコマンドを使用して、現在のレコードを操作したり、レコードに関連するタスクを実行したりします。 使用できるコマンドはレコードの種類によって異なりますが、通常はコマンド バーを使用して、変更の保存、レコードの削除、ページの最新の情報に更新、レコードへのリンクの送信、レコードの所有者の再割り当て、Word テンプレートを使用したレコードのエクスポートができます。
4. **見出しバー**:レコード ビューによっては、見出しバーのレコード名の反対側に、特に重要なフィールドがいくつか表示されます。 これらは通常、現在の種類のレコード (レコード名やレコード所有者など) を操作するための基本的なフィールドです。
5. **すべてのフィールド値の表示と編集**:レコード ビューの本体には、現在のタブ、フォーム ビュー、およびレコードの種類に関連するすべてのフィールドが表示されます。 赤いアスタリスクでマークされたフィールドは必須であり、有効な値を持たないレコードを保存することはできません。 青いプラス記号でマークされたフィールドは、特に重要または推奨されますが、厳密には必須ではありません。 ロック アイコンが表示されるフィールドは読み取り専用で、編集することはできません。

## <a name="record-set-navigation"></a>レコードのセットのナビゲーション 

事前設定されたビューとクエリを使用して、複数のレコード間を移動します。 レコードを使用したナビゲーションでは、作業一覧を失うことなく、一覧のレコード間を移動し、元に簡単に戻ることができるので、ユーザーの生産性が高まります。

## <a name="reference-panel"></a>参照パネル

参照パネルは、現在の画面から移動せずに、作業を行うための優れた方法です。 別の画面に移動することなく、表示しているレコードのコンテキスト内で、ケースやアカウントの営業案件など、他の関連項目を参照できます。

## <a name="notifications"></a>通知 

フォームに表示される通知には、情報、警告、エラーの 3 種類があります。 通知は常にヘッダーの真上のフォームの上部にあります。

エラー通知を選択すると、フォーム上のエラーが発生したフィールドに移動します。

![通知の例](media/notifications.png "通知の例")

凡例:

1. **Info**: 通知は情報提供です。
2. **Warn**: 通知は警告です。 
3. **Error**: 通知はエラーです。 



### <a name="single-notification"></a>単一の通知

通知が 1 つのみの場合は、1 行表示されます。

![単一の通知の例](media/single_notification.png "単一の通知の例")

### <a name="multiple-notifications"></a>複数の通知

複数の通知がある場合は、通知の数が表示されます。 各メッセージを表示するには、シェブロンを選択します。

![複数の通知の例](media/multiple_notification.png "複数の通知の例")
