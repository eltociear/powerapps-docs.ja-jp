#  <a name="basic-navigation-in-a-model-driven-app"></a>モデル駆動型アプリでのナビゲーションの基本 

この概要では、アプリを検索して開く方法と、リスト、フォーム、ビジネスプロセスなどの一般的なユーザーインターフェイス要素を操作する方法について説明します。

## <a name="navigating-among-apps-areas-and-entities"></a>アプリ、区分、およびエンティティ間の移動

モデル駆動型アプリは、アプリケーション (アプリ)、区分、およびエンティティから構築されています。

- *アプリ*には、アカウントや連絡先の管理など、特定のクラスのアクティビティを実現するための機能のコレクションが用意されています。 アプリセレクターメニューを使用して、組織で利用可能なアプリ間を移動します。

- *作業領域*は、特定の機能専用のアプリの部分です。 各作業領域には、その領域で作業するためのエンティティのコレクションが用意されています。 場合によっては、同じエンティティが複数の領域 (または複数のアプリ) に表示されることがあります。 たとえば、contacts エンティティと accounts エンティティは、さまざまなアプリや作業領域に表示されます。 作業領域メニューを使用して、現在のアプリの作業領域間を移動します。

- *エンティティ*は、連絡先やアカウントなど、特定の種類のデータを表します。 エンティティは、エンティティで使用できるフィールドのコレクションを定義する、構造化されたデータ形式を使用します。 各エンティティは、個々のレコードのコレクションで構成されます。 たとえば、contacts エンティティの場合、各レコードには1人の人物が記述され、各レコードには、名、姓、電子メールアドレスなどのフィールドのコレクションが含まれます。 通常、エンティティには2つのビューがあります。リストビューは、通常、使用可能なレコードを一覧表示しています。また、1つのレコードに対して使用可能なすべてのデータと設定を表示するフォームビューです。 現在の作業領域のエンティティ間を移動するには、サイドナビゲーターを使用します。

### <a name="move-between-apps"></a>アプリ間の移動

アプリを切り替えるには、アプリセレクターメニューを使用します。

![アプリセレクターメニュー](media/app-selector.png "アプリセレクターメニュー")

アプリセレクターメニューに表示されるアプリは、アクセスできるアプリによって異なります。 

### <a name="move-between-entities-records-and-work-areas"></a>エンティティ、レコード、および作業領域間の移動

お気に入りのレコードや使用頻度の高いレコードに簡単にアクセスして戻すことができます。 次の図は、プライマリナビゲーション要素を示しています。

![ナビゲーションコントロール、展開ビュー](media/nav-expanded.png "ナビゲーションコントロール、展開ビュー")

文字列

1. **アプリセレクター**: アプリ間を移動するには、このメニューを開きます。
1. **折りたたみ/展開ボタン**: このチェックボックスをオンにすると、ページのメイン部分のスペースを広げることができます。 ナビゲーターが既に折りたたまれている場合は、このボタンを選択してもう一度展開します。
1. [最近使った**レコード**]: このエントリを展開すると、最近使用したレコードの一覧が表示されます。 ここでレコードを選択して開きます。 ここに一覧表示されているレコードの横にあるプッシュピンアイコンを選択して、お気に入り (固定されたレコード) に追加します。
1. **お気に入りのレコード**: お気に入りの (ピン留めされた) レコードを表示して開くには、このエントリを展開します。 [最近使った**レコード**] の一覧を使用して、ここにレコードを追加します。 この一覧から削除するには、ここに表示されているレコードの横にある [削除] アイコンを選択します。
1. **[エンティティナビゲーター]** : この領域には、現在の作業領域で使用できる各エンティティとダッシュボードが一覧表示されます。 ここで任意のエントリを選択して、そのエンティティの名前付きダッシュボードまたはリストビューを開きます。
1. **作業領域セレクター**: 別の作業領域に移動するには、このメニューを開きます。 現在の作業領域には、ここで名前が付けられています。

## <a name="working-with-list-views"></a>リストビューの操作

通常、最初にエンティティを開いたときに、リストビューが表示されます。このビューには、そのエンティティに属するレコードの一覧が表形式で表示されます。 たとえば、 **accounts**エンティティを開くと、アカウントの一覧が表示されます。

![一般的なリストビュー](media/list-view.png "一般的なリストビュー")

文字列

1. **レコードの選択**: この列にチェックを配置して1つ以上のレコードを選択します。 作業中の場所に応じて、コマンドバーのボタンを使用して、選択したすべてのレコードに一度に1つの操作を適用できます。
2. **レコードを開く**: 一覧内の任意のレコードを選択すると、レコードビューが開き、レコードの詳細がすべて表示されます。 通常、現在のエンティティからレコードを開くには、 **[名前]** 列から選択する必要があります。 一部のエンティティは、他の列 (関連する連絡先など) の関連エンティティからのレコードへのリンクを提供します。
3. [**一覧の並べ替えまたはフィルター**選択]: 選択した列の値で一覧を並べ替えるか、その列の値で一覧をフィルター処理します。 列見出しの矢印は、どの列がどの方向に並べ替えられているかを示します。 
4. **コマンドバー**: コマンドバーのコマンドを使用して、一覧のレコードを操作し、関連するアクションを実行します。 一部のコマンド (**削除**など) では、最初に1つ以上のターゲットレコードを選択する必要があります。一番左の列にチェックマークを配置し、他のコマンドはリスト全体に対して動作します。 使用するレコードの種類に応じて、一覧を Excel ブックにエクスポートしたり (テンプレートに基づいて)、グラフやダッシュボードを開いたりすることができます。
5. **[ビューを検索**する]: 一覧の上にある [検索] フィールドにテキストを入力すると、現在のビュー内のテキストを含むレコードのみが表示されます。
6. **フィルターとページング**: 文字を選択すると、その文字で始まる名前を持つレコードのみが表示されます。 一覧に、1ページに表示できるよりも多くのレコードが含まれている場合は、一覧の下部にあるページング矢印を使用して、ページを前後に移動します。

## <a name="working-with-record-views"></a>レコードビューの操作

レコードビューには、1つのレコードに関するすべての詳細情報が表示されます。また、そのレコードを操作するための特別な機能を提供することもあります。 通常は、リストビューに表示されるレコードを選択してレコードビューを開きますが、関連レコードからのリンクに従ってレコードビューを開くこともできます。

![一般的なレコードビュー](media/form-view.png "一般的なレコードビュー")

文字列

1. **タブ**: ほとんどのレコードビューはタブに分かれています。各タブには、レコードから関連するフィールドのコレクションが用意されています。 タブが使用可能な場合は、レコード名の下に一覧表示されます。 タブ名を選択して、そのタブにアクセスします。現在のタブは下線付きで表示されます。
2. **関連**: ほぼすべての種類のレコードは、少なくとも1回保存した後に**関連**するタブとして表示されます。 このタブは、実際には、表示されているレコードを使用または参照する他の種類のレコードを検索するために使用できるドロップダウンリストです。 **関連**するドロップダウンリストからエンティティ名を選択すると、そのエンティティに対してという名前の新しいタブが開き、その種類のすべての関連レコードの一覧が表示されます。 **[関連]** タブは引き続き使用でき、現在のタブを参照している他の種類のレコードを検索する場合にも使用できます。
3. **コマンドバー**: コマンドバーのコマンドを使用して、現在のレコードを操作したり、レコードに関連するタスクを実行したりします。 使用できるコマンドはレコードの種類によって異なりますが、通常はコマンドバーを使用して変更を保存したり、レコードを削除したり、ページを更新したり、レコードへのリンクを電子メールで送信したり、レコードの所有者を再割り当てしたり、Word テンプレートを使用してレコードをエクスポートしたりできます。
4. **見出しバー**: いくつかのレコードビューでは、レコード名とは逆に、見出しバーに特に重要なフィールドがいくつか表示されます。 これらは通常、現在の種類のレコード (レコード名やレコード所有者など) を操作するための基本的なフィールドです。
5. **すべてのフィールド値を表示および編集**する: レコードビューの本文では、現在のタブ、フォームビュー、およびレコードの種類に関連するすべてのフィールドが検索されます。 赤いアスタリスクでマークされたフィールドは必須であり、有効な値を持たないレコードを保存することはできません。 青いプラス記号でマークされたフィールドは、特に重要または推奨されますが、厳密には必須ではありません。 ロックアイコンが表示されているフィールドは読み取り専用で、編集することはできません。

## <a name="notifications"></a>通知 

フォームに表示される通知には、"情報"、"警告"、および "エラー" の3種類があります。 通知は常にヘッダーの真上のフォームの上部にあります。

エラー通知を選択すると、エラーが発生したフォーム上のフィールドに移動します。

![通知の例](media/notifications.png "通知の例")

文字列

1. **情報**: 通知は情報提供です。
2. **警告**: 通知は警告です。 
3. **エラー**: 通知はエラーです。 


### <a name="single-notification"></a>1回の通知
通知が 1 つのみある場合は、行が 1 つ表示されます。

![シングル通知の例](media/single_notification.png "シングル通知の例")

### <a name="multiple-notifications"></a>複数の通知
通知が 2 つ以上ある場合は、通知の数が表示されます。 各メッセージを表示するには、シェブロンを選択します。

![複数の通知の例](media/multiple_notification.png "複数の通知の例")
